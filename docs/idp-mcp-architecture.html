Detailed Request Flow — IDP Assistant Backend

1
Frontend sends POST with question
React page calls fetchApi with JSON body
POST /api/idp-assistant/query { "question": "..." }
2
Backend opens SSE stream
Sets Content-Type: text/event-stream, flushes headers immediately so the browser starts receiving data
res.setHeader('Content-Type', 'text/event-stream') → res.flushHeaders()
3
Resolve internal service URLs
Uses Backstage Discovery API to get internal cluster URLs and mints a service-to-service auth token for the catalog
discovery.getBaseUrl('catalog') • discovery.getBaseUrl('cost')
4
Start Claude streaming (Turn 1…N, max 10)
Calls Anthropic API with the question, system prompt, and tool definitions. As Claude writes text, each chunk is immediately emitted as an SSE event
anthropic.messages.stream({ model: 'claude-sonnet-4-6', tools: TOOLS, ... })
5
Claude requests tool execution
If Claude's response contains tool_use blocks, the backend emits a tool_start event so the UI shows a spinner chip, then executes the tool locally
data: {"type":"tool_start","name":"get_cost_summary"}
6
Backend calls Backstage API and returns result to Claude
For catalog tools: uses internal URL + bearer token. For cost tools: calls cost plugin directly (unauthenticated). Result is added to the message thread as tool_result
fetch(`${catalogUrl}/entities?filter=kind=Resource`) → append to messages[]
7
Claude streams final answer & SSE closes
Claude writes a human-readable answer using the tool results. Each text chunk streams to the browser. When Claude stops calling tools, the loop ends and the stream is closed
data: {"type":"text","content":"..."} → data: {"type":"done"}

Available Tools

list_environments
List all environment/namespace resources registered in the IDP catalog
no params
get_environment_resources
Get all infra resources (Cloud SQL, GCS, Redis) provisioned for an environment
environment (req)
list_applications
List all application components deployed to a specific environment
environment (req)
list_systems
List all systems (network codes / cost centers) in the catalog
no params
get_entity
Get full details of any catalog entity by kind and name
kind (req) name (req)
get_cost_summary
Current month GCP spend grouped by network code — ideal for a quick overview
no params
get_cost_trends
Monthly GCP cost history for the last N months, grouped by network code
months (opt, default 3)
get_cost_breakdown
Current month GCP spend broken down by service (K8s, Cloud SQL, GCS…) for one network code
network_code (req)
Catalog API (auth token required) Cost Insights API (unauthenticated) ■ optional param ■ required param
